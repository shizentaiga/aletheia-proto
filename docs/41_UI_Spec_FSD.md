# エリア選択 UI/UX 仕様書 (Full-screen Stack Drilldown) Ver 0.8 [MVP確定版]

## 1. コンセプト
「5/1 安定稼働」を最優先とし、バグの温床となる複雑なJSロジックを排除。
HTML Fragment (Hono + HTMX) を主軸とした「枯れた技術」の組み合わせで、ネイティブ風の操作感を実現する。

## 2. ミニマム階層設計
### 2.1 階層の制限
複雑なデータクレンジングを避けるため、以下の3階層に限定する。
- **Level 1**: 地方（関東, 関西...）
- **Level 2**: 都道府県（東京都, 神奈川県...）
- **Level 3**: 市区町村（新宿区, 渋谷区...）
※ Level 4 (町域) は将来的な検討項目として削除。

### 2.2 通信方式の簡素化
JSON API ＋ クライアントサイド・レンダリングを廃止し、HTMXによるHTML断片の取得に一本化する。
- **Request**: GET `/api/area-selection?parent_id=13` (HTMX `hx-get`)
- **Response**: `<li>...</li>` のリスト（HTML Fragment）
- **キャッシュ**: `Cache-Control: public, max-age=3600` を付与し、Cloudflare層でのキャッシュを優先。

## 3. インタラクション & 履歴管理（安全性の強化）
### 3.1 履歴スタックに依存しない開閉制御
`history.back()` による予期せぬページ遷移を回避するため、ハッシュの直接書き換えによる制御を行う。
- **展開時**: `location.hash = 'area';` を付与。
- **閉じる時（確定・キャンセル共通）**: `history.replaceState(null, '', location.pathname + location.search);` によってハッシュを消去する。
- **popstate 監視**: `window.onpopstate` を監視し、物理戻るボタンでハッシュが消えた際も、JS側で確実に閉じるアニメーションを発動させる。

### 3.2 アニメーションとブラウザ制御
- **方式**: `translateX` を利用したスライド。
- **デフォルト操作の抑制**: CSSにて `touch-action: none;` および `overscroll-behavior-x: none;` を指定し、ブラウザのスワイプ戻りによる画面ハングを防止する。

### 3.3 物理的ユーザビリティ
- **戻るボタン**: 左上の「戻る」ボタンのタップ領域を `48px` 以上確保。
- **親指圏内の操作**: 画面下部にも「決定/閉じる」ボタンを配置し、操作の完結を助ける。

## 4. 運用・パフォーマンス（MVP）
- **クエリ**: D1 への生クエリで実行。集計キャッシュ (4.2) は 5/1 以降に検討。
- **スクロール**: 階層切り替え時は一律 `scrollTo(0, 0)`。
