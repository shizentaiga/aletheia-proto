# エリア選択 UI/UX 仕様書 (Full-screen Stack Drilldown) Ver 0.6 [MVP版]

## 1. コンセプト
「5/1 安定稼働」を最優先とし、バグの温床となる複雑なJSロジックを排除。
HTML Fragment (Hono + HTMX) を主軸とした「枯れた技術」の組み合わせで、ネイティブ風の操作感を実現する。

## 2. ミニマム階層設計
### 2.1 階層の制限
複雑なデータクレンジングを避けるため、以下の3階層に限定する。
- **Level 1**: 地方（関東, 関西...）
- **Level 2**: 都道府県（東京都, 神奈川県...）
- **Level 3**: 市区町村（新宿区, 渋谷区...）
※ Level 4 (町域) は将来的な検討項目として削除。

### 2.2 通信方式の簡素化
JSON API ＋ クライアントサイド・レンダリングを廃止し、HTMXによるHTML断片の取得に一本化する。
- **Request**: `GET /api/area-selection?parent_id=13` (HTMX `hx-get`)
- **Response**: `<li>...</li>` のリスト（HTML Fragment）
👉 **利点**: クライアント側で `map()` やテンプレート生成を行う必要がなくなり、Vanilla JS の責務が「アニメーション制御」のみに限定される。

## 3. インタラクション & 履歴管理（簡素化）
### 3.1 履歴管理の最小実装
複雑な `pushState`/`replaceState` の同期を廃止し、ブラウザの標準挙動に委ねる。
- **展開時**: URLにハッシュ（例: `#area`）を付与。
- **閉じる時**: `history.back()` を実行、またはハッシュを削除。
- **内部遷移**: JSの状態変数で現在の階層を管理。履歴には刻まない。

### 3.2 アニメーション
- **方式**: `translateX` を利用したスライド。
- **実装**: Vanilla JS は CSS クラスの付け替え（`open` / `active`）のみを担当。

## 4. 運用・パフォーマンス（MVP）
- **クエリ**: D1 への生クエリで実行。集計キャッシュ (4.2) は 5/1 以降に検討。
- **スクロール**: 階層切り替え時は一律 `scrollTo(0, 0)`。厳密な位置復元 (5.1) は行わない。

## 5. 【将来的な検討項目】（Backlog）
MVPリリース後のアップデート、またはリソース確保後に検討する項目群。

| 項目 | 内容 | 導入のメリット |
| :--- | :--- | :--- |
| **Level 4 拡張** | 町域レベル（西新宿等）の検索 | よりピンポイントな探索が可能になる |
| **エッジスワイプ** | 左端スワイプでの戻る操作 | iOS ネイティブに近い操作感の提供 |
| **高度な履歴管理** | 階層ごとの `pushState` 同期 | ブラウザ戻るボタンの完全な直感化 |
| **集計キャッシュ** | `area_stats_cache` テーブルの導入 | D1 クエリ負荷の軽減と高速化 |
| **精密スクロール復元** | クエリハッシュによる位置保持 | 前の階層に戻った際の再探索コスト低減 |

## 6. 技術スタックの責務再定義（MVP版）

| 技術 | 役割 |
| :--- | :--- |
| **Hono** | HTML Fragment の生成、D1 生クエリの実行 |
| **HTMX** | `hx-get` によるリストの部分書き換え、ローディング表示 |
| **Vanilla JS** | レイヤーの `transform` アニメーション、ハッシュ監視 |
