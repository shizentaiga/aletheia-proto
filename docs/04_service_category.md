# 04_Service_Category: サービス設計・カテゴリ定義（v1.1）

## 1. 目的：なぜカテゴリを分類するのか
本プロジェクトにおいて、カテゴリ分類は単なるラベル付けではなく、**「ユーザーの現在の動機（心のモード）」と「最適な地点」を最短距離で結ぶためのナビゲーション・インフラ**として機能します。
認知負荷を最小限に抑え、直感的なタップ操作だけで目的の場所に到達できる体験を設計します。

---

## 2. カテゴリ体系：動詞による「知覚的入り口」
効率性ではなく、ユーザーがその場所で「何をするか（コト）」という動詞に基づいた8つの大分類を策定します。

| カテゴリ名（UI） | サブ・コピー（動詞） | 包含内容の例 |
| :--- | :--- | :--- |
| **1. 飲食・グルメ** | 食を楽しむ | カフェ、レストラン、バー、直売所 |
| **2. 美容・健康** | 身体を整える | 美容室、整体、ヨガ、カウンセリング |
| **3. ビジネス・仕事** | 価値を創る | コワーキング、コンサルティング、士業、求人 |
| **4. 学び・自己研鑽** | 知性を磨く | コーチング、英会話、塾、占い |
| **5. 旅行・レジャー** | 非日常を味わう・祈る | 宿泊、体験、神社・仏閣、記念撮影 |
| **6. ファッション** | 自分を装う | アパレル、パーソナルカラー診断、眼鏡 |
| **7. 住まい・空間** | 環境を整える | 不動産、建築、内装、クリーニング |
| **8. 生活サポート** | 日常を回す | 修理、ペットホテル、家事代行 |

---

## 3. フェーズ別展開戦略

### Phase 0: 密度の検証（カフェ特化）
「飲食・グルメ（食を楽しむ）」カテゴリに全リソースを集中。
小岩・田端等の特定エリアにおいて、個人カフェやチェーン店の情報を網羅的に集約し、情報の「密度」がもたらすセレンディピティ（偶然の発見）を検証します。

### Phase 1: 予約・決済のテスト
「美容・健康」および「ビジネス・仕事」カテゴリを解放。
ヨガ講師のレッスン予約や、経営コンサルタント等の専門サービスにおいて、対人支援の「時間枠管理（Slots）」と「決済」の動線をテストします。

### Phase 2: 全カテゴリ解放と外部誘導
物販（アパレル等）は外部サイトへの誘導に特化し、自社システムは「時間の予約（サービス提供）」という対人インフラとしての価値に純化させます。

---

## 4. 実装の設計指針

* **Primary Category（単一保持）**:
    * 検索速度の極大化のため、主カテゴリは「1サービスにつき1つ」に限定（Integer型でDB保持）。
* **「コト（予約）」への特化**:
    * 在庫管理の負荷を排除するため、物販ではなく「対人支援・空間提供」の予約システムとして最適化します。
* **属性タグ（Metadata）**:
    * カテゴリに依存しない横断的な属性（Wi-Fi、電源、禁煙等）は、`service_cafe_details` のような拡張テーブル、またはタグ形式で柔軟に保持します。

---

## 5. 結論
本設計は、8つの動詞による壮大な分類学（タクソノミー）を背景に持ちながら、実装においては「近所のカフェ」という最も身近な接点から秩序を形成し始めるものです。
この「大局的に見て、小局から始める」アプローチにより、開発リソースを分散させることなく、ALETHEIAが目指す「隠れた価値との出会い」を、確実かつ迅速なユーザー体験として提供します。
