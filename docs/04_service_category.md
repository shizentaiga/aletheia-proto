# サービスカテゴリ分類 妥当性報告書

**策定日：2026年3月30日**

---

## 1. 目的：なぜカテゴリを分類するのか
本報告書は、検索・閲覧・予約・決済プラットフォーム『 』の開発において、 **「検索条件の利便性と、摩擦のないユーザー体験」** を設計するために作成されました。
膨大なサービス群の中から、ユーザーが自身の「現在の動機」に合致する対象を、認知負荷を最小限に抑えて発見・到達できるインフラを構築することが、本分類の主目的です。

## 2. カテゴリ定義：動詞による「知覚的入り口」
ユーザーの「心のモード（ハレ／ケ）」に同期し、直感的な選択を可能にします。将来的には以下の8分類を網羅する設計とします。

| カテゴリ名（UI） | サブ・コピー（動詞） | 包含内容の例 |
| :--- | :--- | :--- |
| **1. 美容・健康** | 身体を整える | 美容室、整体、ヨガ、カウンセリング |
| **2. 飲食・グルメ** | 食を楽しむ | レストラン、カフェ、バー、直売所 |
| **3. ファッション** | 自分を装う | アパレル、パーソナルカラー診断、眼鏡 |
| **4. 住まい・空間** | 環境を整える | 不動産、建築、内装、ハウスクリーニング |
| **5. 学び・自己研鑽** | 知性を磨く | 塾、コーチング、英会話、占い |
| **6. 旅行・レジャー** | 非日常を味わう・祈る | 宿泊、体験、神社・仏閣、撮影（記念） |
| **7. 生活サポート** | 日常を回す | 修理、クリーニング、ペットホテル、家事 |
| **8. ビジネス・仕事** | 価値を創る | コワーキング、士業、求人、コンサルティング |

## 3. フェーズ0：スモールスタート戦略
開発初期においては、以下の3〜4カテゴリに厳選してリリースし、実社会での「予約・決済」の動線をテストします。

1.  **飲食・グルメ（食を楽しむ）**:
    * 近所のカフェを網羅的に登録し、情報の集約性と閲覧性を検証する。
2.  **美容・健康（身体を整える）**:
    * ヨガ講師や整体師の予約サイトとして機能させ、対人支援の「時間枠管理」をテストする。
3.  **ビジネス・仕事（価値を創る）**:
    * 経営コンサルタント等の予約・決済を導入し、高単価・専門的サービスの運用耐性を確認する。

## 4. 運用および実装の設計指針
* **Primary Categoryの単一保持（厳守）**:
    DB負荷抑制と検索速度の極大化のため、主カテゴリ（Integer型）は「1サービスにつき1つ」に限定して設計します。
* **「コト（予約）」への純化と外部誘導**:
    物販（アパレル等）は外部サイトへのURL誘導に特化し、自社システムでの在庫管理負荷を排除します。予約・決済機能は、個人セラピストやコーチ等の「時間の予約」という対人支援インフラに最適化します。

## 5. 結論
本設計は、8つの動詞による壮大なタクソノミー（分類学）を背景に持ちながら、実装においては「近所のカフェ」という最も身近なカオスから秩序を形成し始めるものです。この段階的なアプローチにより、開発リソースを分散させることなく、 が目指す「隠れた才能との出会い」を、確実かつ爆速な体験としてユーザーへ届けることが可能となります。
