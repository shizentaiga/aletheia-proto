# STARBUCKS API Sync & D1 Integration Report

## 1. 概要
本資料は、スターバックス公式サイトの店舗検索APIを介して、全国の店舗データ（約1,900件）を自動取得し、Cloudflare D1 データベースへ正規化されたデータとして統合したプロセスを記録するものである。

## 2. データ取得の技術的背景
### 2.1 従来の課題
- 公式サイトのUI上では、スクロールに応じて100件ずつ順次読み込まれる仕様であり、全県・全店舗の情報を俯瞰的に把握・抽出することは困難であった。

### 2.2 APIの特定（リバースエンジニアリング）
- ブラウザのデバッグモード（Networkタブ）を用いた調査により、以下のAPIエンドポイントを特定。
  - **Endpoint**: `https://hn8madehag.execute-api.ap-northeast-1.amazonaws.com/...`
  - **Format**: JSON（構造化データ）
  - **Security**: 直接のURLアクセスは `{"message":"Forbidden"}` となる。リクエスト時に `Origin` および `Referer` を含むブラウザのコンテキストを正しくセットする必要がある。

## 3. 実装と自動化
### 3.1 取得スクリプト (fetch_and_convert_stb.js)
- **並列処理の制御**: サーバー負荷を考慮し、リクエスト間に **3秒の待機時間 (Wait)** を設定。
- **ページネーション**: `found`（総件数）を動的に解析し、100件を超える自治体（東京都等）も自動的にループ処理を実施。
- **カバレッジ**: 都道府県コード `01`〜`47` を一括走査。

### 3.2 処理結果
- **総取得件数**: 1,887件
- **生成SQL**: `INSERT OR REPLACE` 形式（冪等性の担保）

## 4. D1 への投入における留意点
### 4.1 トランザクション管理の自動化
- **エラー事象**: SQLファイル内に `BEGIN TRANSACTION;` および `COMMIT;` を記述した際、D1側のトランザクション管理機構と衝突しエラーが発生した。
- **解決策**: D1の `execute --file` 命令は、ファイル全体を自動的に一つのトランザクションとして処理するため、明示的な開始・終了命令は不要である。

## 5. データの資産価値
取得した `st_all_raw.json` は生データとして保持。今後のスキーマ変更や、営業時間・設備フラグ（Wi-Fi、リザーブ店等）の追加要求に対し、再リクエストなしで迅速に対応可能な状態となっている。
