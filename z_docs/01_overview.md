# プロジェクト『Aletheia（アレテイア）』システム要件定義書（Ver. 3.5）

**キャッチコピー：つながりは偶然から**

---

## 1. ビジョンと倫理（Core Ethics）
* **開放されたプラットフォーム**: 世界中のすべての人を対象とした、ボーダレスな「つながりの基盤」を目指す。
* **脱・囲い込み**: データ所有権はユーザーに帰属。登録データはCSV等で持ち出し可能とする。

## 2. UI/UXプロトコル（One-Column & Flexible Routing）
* **タップ・ファーストUI**: 直感的なチップス（タグ）選択を主導線とし、文字入力は補助機能とする。
* **段階的な編集設計**: 初期は「閲覧URL」と「編集URL」を分離し、実装の確実性を優先する。
* **動的ルーティング**: サービスドメイン（決定後）に基づき、 `/p/[id]` 形式の固有URLを発行。

## 3. 検索・評価ロジック（Trust & Neighborhood）
* **統合型ランキング**: 1日最大100、直近30日で最大3,000カウントを初期評価基準とし、新陳代謝を促す。
* **近傍リコメンド（Neighborhood）**:
    * **完全非公開**: 「お気に入り」データは内部処理のみに使用し、第三者には一切公開しない。
    * **共鳴の提示**: お気に入りリストの類似性が高いユーザー（数学的近傍）の間で支持されている対象を、優先的に「おすすめ」として提示する。
* **最適化**: 運用データに基づき、数値およびロジックは事前告知（重大変更時）の上で見直しを行う。

## 4. テクニカル・ガバナンス（Technical Debt Defense）
* **言語・インフラ**: TypeScript / Cloudflare Workers / D1 / R2。
* **拡張性を担保する最小実装（出戻り防止策）**:
    1.  **認証予約**: 初期D1スキーマに `owner_google_id` (NULL許容) を定義し、完全認証移行への接合部を確保。
    2.  **非同期集計**: Log収集と Cron Trigger（日次バッチ）により、ランキングと近傍演算を更新する。
    3.  **移行トリガー**: 登録者数100件超過、またはレイテンシ増大時に D1（DB）へ完全移行する。

## 5. 収益モデル（Clear Boundaries）
* **基本無料**: 検索・閲覧・外部URLへの誘導（リンク形式）は永続無料。
* **予約・決済課金**: システム内の「予約・決済機能」を利用する場合は、一律有料プランの対象とする。

## 6. 免責と法遵守
* **責任範囲**: システムトラブルによるデータ損失等は過失なき限り免責。個人情報保護法に基づき適正に運用する。
