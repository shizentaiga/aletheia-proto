# 11_Technical_Notes (主要技術選定の理由)

プロジェクト『Aletheia』を支える、高速で壊れにくい技術スタックの採用理由です。

* **TypeScript**: 実行前にエラーを検知し、属人化を防いでシステムの堅牢性を担保するため。
* **Hono**: 標準Web技術に準拠し、エッジ環境で世界最速級のレスポンスを実現するため。
* **React (Hono JSX)**: 画面を部品化して再利用性を高め、UIの変更や拡張を安全に行うため。
* **Cloudflare Workers**: サーバー管理を不要にし、低コストかつグローバルな配信基盤を得るため。
* **Cloudflare D1 (SQLite)**: サーバーレス環境と親和性が高く、構造化データを高速に処理するため。
* **Cloudflare R2**: 画像や大容量アセットを、配信コストを抑えつつ安全にストレージするため。
* **Trie木構造 (Hono内部)**: URLが増えても高速なルート探索を維持し、ユーザーを待たせないため。
* **Single-column UI**: モバイル操作に特化し、ユーザーの認知負荷を最小限に抑える設計思想。
* **NULL許容 ID設計**: 開発初期のスピードを優先しつつ、将来のGoogle認証連携への道を塞がないため。
* **Cron Trigger**: ランキングやリコメンドの演算を自動化し、鮮度の高い「つながり」を維持するため。
