# ALETHEIA（アレテイア）

> **「つながりは、偶然から。個人の記録を起点とした地点検索プラットフォーム」**

ALETHEIAは、効率性や画一的な検索アルゴリズムに依存しない、直感的な地点発見体験を提供します。
Google Maps APIを「初期の発見ツール」として活用し、ユーザーの熱量（記録・編集）を通じてデータを「自社資産」へと昇華させる、自律型データベース・プラットフォームです。

## 2. プロジェクト構造と役割：実行境界の厳格な分離

「どこで動いているコードか」を物理的に判別し、ランタイムエラーとスパゲッティ化を防止します。

### ■ アプリケーション・コア (`src/`)

| フォルダ / 拡張子 | 役割 (責務) | 実行環境 | 不具合時の調査対象 |
| :--- | :--- | :--- | :--- |
| `*.server.ts` | **機密・DB操作**。D1, Bindings, 認証。 | Server | データ欠如、SQLエラー、500エラー |
| `*.client.ts` | **動的・Fetch**。イベント, UI更新。 | Browser | ボタン無反応、画面更新不良 |
| `*.ui.tsx` | **純粋な見た目**。ロジック抜きのJSX。 | Shared(SSR) | レイアウト崩れ、表示ミス |
| `services/` | データアクセス層（D1等との疎通）。 | Server | DBクエリロジック、データ整合性 |
| `hooks/` | フロントエンド論理（状態管理）。 | Browser | UIの状態遷移、非同期通信管理 |
| `types/` | データの法律。型定義(Zod)とSchema。 | Shared | 型不一致、バリデーション失敗 |

## 3. データの成長モデル (Data Evolution)

本プロジェクトのコア・コンセプトです。データは以下のフェーズを経て「自律」します。

1. **初期シード期 (Level 0)**: 外部APIより最小限の情報を取得（名称・住所のみ）。
2. **データ昇華期 (Level 1)**: 有料ユーザー（Editor Pro）やオーナーによる「確定・修正」アクションにより、**自社資産**として定義を上書き。
3. **自律稼働期 (Level 2)**: 特定の基盤に依存しない、ALETHEIA独自の高密度データベースとして運用。

## 4. 開発フロー：三段階のステップ

1. **[思考] Sandbox**: `src/_sandbox/` でロジックの断片やAPI単体テストを試作。
2. **[検証] Proto (本リポジトリ)**: 正規フォルダ（`src/services`等）に組み込み、システム整合性を確認。
3. **[昇格] Production**: 検証済みのコードを本番用リポジトリへ移植。

---

## 5. デバッグと疎通の掟 (The Rule of Traceability)

### ① 最小経路の優先 (Atomic First)
複雑なHookを作る前に、必ず API エンドポイント単体でDBとの疎通を確認すること。画面が動かない時は、まずブラウザで直接 `/api/xxx` を叩き、生データを確認すること。

### ② ログの指紋 (Log Signatures)
調査場所を即座に特定するため、ログに接頭辞を付ける。
- `console.log("[Server:API] ...")` -> サーバー・DB側の問題（Wranglerログを確認）
- `console.log("[Client:Hook] ...")` -> ブラウザ側の問題（DevToolsを確認）

### ③ 実行境界の死守
`D1Database` 等のサーバー専用オブジェクトを `*.client.ts` に持ち込まない。ブラウザは常に API を経由する「疎結合」を維持すること。

---

## 6. セキュリティ・ポリシー
- **秘匿情報の隔離**: APIキーやシークレットは `wrangler secret put` で管理。
- **バリデーションの二重化**: 静的な `TypeScript` で開発ミスを防ぎ、動的な `Zod` で不正データからシステムを守る。

## 📂 ディレクトリ構成
現在のプロジェクトは、役割ごとに「基盤・表示・データ・意匠」の4層に整理されています。

```
├── scripts/                # 外部データ取得・変換パイプライン
└── src/
    ├── index.tsx           # エントリーポイント（ルーティング定義）
    ├── renderer.tsx        # 共通レイアウト基盤（HTMLメタ・アセット管理）
    ├── api_handlers.tsx    # 司令塔（データ取得とレンダリングの仲介）
    │
    ├── components/         # 【汎用UI】再利用可能な最小単位のコンポーネント
    │   ├── CafeCard.tsx    # 店舗カード
    │   ├── DebugMonitor.tsx # 開発用パネル
    │   ├── SearchHeader.tsx # 検索結果件数表示
    │   ├── SearchLogic.tsx  # クライアントJS注入用コンポーネント
    │   └── SearchScripts.ts # JSロジック本体
    │
    ├── pages/              # 【画面構成】機能・ページ単位のディレクトリ
    │   └── Top/            # Topページ専用
    │       ├── TopPage.tsx    # ページ全体の組み立て（司令塔）
    │       ├── TopHeader.tsx  # ナビゲーション
    │       ├── TopSearch.tsx  # 検索フォーム
    │       ├── TopList.tsx    # リスト・追加読み込みロジック
    │       ├── TopStyles.tsx  # ページ専用CSS
    │       ├── TopConfig.ts   # デザイン定数・UI文言
    │       └── TopScripts.tsx # UI同期スクリプト
    │
    ├── db/                 # 【データアクセス層】

    │   ├── schema.sql      # テーブル定義
    │   └── seed/           # マスターデータ（core / development / 地域・チェーン別）
    │
    ├── lib/                # 【共通基盤】

    │
    └── styles/             # 【意匠層】
        └── theme.ts        # デザインシステム（Color/Space/Font）
```
