## 1. 開発・運用コマンド
ターミナル（コンソール）で使用する主要な操作コマンドです。
- `npm run dev`: **ローカル開発サーバー起動** 。PC内で変更を即座に確認できます。
- `npm run deploy`: **Cloudflare本番公開** 。世界中に最新版を配信します。

## 2. プロジェクト構造と役割

### ■ システム基盤
- `wrangler.jsonc`: **Cloudflare配信設定** 。公開名や、`/public` フォルダをルートとして扱う定義。
- `package.json`: **依存関係・バージョン管理** 。使用するライブラリ（Hono等）の目録。
- `tsconfig.json`: **TypeScript設定** 。コードの書き間違いをチェックするルールの定義。

### ■ サーバーサイド（src/）
部品化（コンポーネント化）を徹底し、保守性を高めています。
- `src/index.tsx`: **アプリケーションの司令塔（ルーティング）** 。どのURLでどの部品を表示するかを制御。
- `src/components/Layout.tsx`: **共通外観パーツ** 。全ページ共通のHTML構造を担当。
- `src/components/Hero.tsx`: **検索インターフェース** 。独立した検索窓の部品。
- `src/components/DevCore.tsx`: **メイン機能実装パーツ** 。絞り込みチップやカレンダー表示を担当。

### ■ 静的資産・データ（public/）
エンジニアでなくても修正・更新が可能な領域です。
- `public/style.css`: **デザイン定義** 。色、余白、レイアウトを一括管理。
- `public/reserve.json`: **マスターデータ** 。予約枠やサービス内容。将来のDB化への雛形。
- `public/client-logic.js`: **ブラウザ動作ロジック** 。検索実行やデータの流し込みを制御。

### ■ システム基盤・隠しファイル（GitHubに送らないファイル！）
- `.gitignore`: **Git除外設定**。`node_modules` や環境変数など、GitHubに送らないファイルを定義。
- `.wrangler/`: **Wranglerキャッシュ**。開発ツールが一時的に使用するフォルダ（同期不要）。
- `.DS_Store`: **Mac専用ファイル**。フォルダの表示設定。リポジトリには不要。

## 3. 設計思想：メンテナンス性の確保
本プロジェクトでは、 **「構造（TSX）」「見た目（CSS）」「データ（JSON）」「動き（JS）」を分離** しています。

UIパーツを「Hero（検索）」と「DevCore（表示・予約）」に分割することで、以下のメリットを享受します：

1. **修正の局所化**: 検索窓だけを変えたい時は `Hero.tsx` を触るだけで済み、他への影響を最小限に抑えます。
2. **安全な拡張**: 既存のパーツ（Layout等）を再利用しながら安全に機能追加が行えます。
3. **分業の容易化**: 非エンジニアによるデータ更新（JSON/CSS）と、エンジニアによるシステム改善を並行して行えます。
